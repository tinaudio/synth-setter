# Shard Metadata

`ShardMetadata` is the sidecar JSON written into wds tar shards (member
`metadata.json`). It mirrors the `RenderConfig` fields the HDF5 layout
stores as `audio` dataset attrs and serves as a trust boundary for
JSON read off R2. The autogenerated field list below is the source of
truth — refer to it for the complete schema.

::: synth_setter.pipeline.schemas.shard_metadata.ShardMetadata

`BlobFieldSpec` is the per-column shape/dtype of a Lance `large_binary` (BLOB)
column (`audio` / `mel_spec`), embedded in the shard's schema metadata so the
reader and validator can recover the row geometry without the render config. It
is read off a shard file, so it is also a trust boundary.

::: synth_setter.pipeline.schemas.shard_metadata.BlobFieldSpec
